## Overview

Edgit supports multiple component types to categorize and organize your code. Built-in types include functions, agents, configs, and SQL queries, plus you can define custom types.

## Built-in Component Types

### Template ðŸŽ¨

HTML/Email/PDF templates for web applications and communications.

**Use For**:
- Email templates (welcome emails, invoices, newsletters)
- HTML page templates (dashboards, reports, landing pages)
- PDF document templates (invoices, reports, certificates)
- Reusable UI components (headers, footers, cards)
- Multi-format responsive designs (MJML, Handlebars, HTML)

**Registration**:
```bash
edgit components add welcome-email templates/email/welcome.html template
edgit components add invoice templates/pdf/invoice.html template
edgit components add header templates/components/header.html template
```

**Common Metadata**:
```json
{
  "type": "template",
  "path": "templates/email/welcome.html",
  "description": "Welcome email for new users",
  "metadata": {
    "engine": "handlebars",
    "format": "html",
    "category": "email",
    "responsive": true,
    "variables": ["name", "appName", "activationUrl"],
    "usedBy": ["send-welcome-member"],
    "preview": "templates/email/welcome.preview.png"
  }
}
```

**Example Component**:
```html templates/email/welcome.html
<!DOCTYPE html>
<html>
<head>
  <title>Welcome to {{appName}}</title>
</head>
<body>
  <h1>Welcome, {{name}}!</h1>
  <p>Thanks for joining {{appName}}. Click below to activate your account:</p>
  <a href="{{activationUrl}}">Activate Account</a>
</body>
</html>
```

**Conductor Integration**:
Templates integrate seamlessly with [Conductor's HTML, Email, and PDF members](/conductor/member-types/html). Reference templates using the `template://` protocol:

```yaml
members:
  - name: send-welcome
    type: Email
    config:
      template: template://email/welcome@v1.0.0
      data:
        name: ${input.userName}
        appName: "MyApp"
```

**Why Templates in Edgit?**
- **Instant updates**: Deploy template changes without rebuilding your application
- **Version control**: A/B test different template versions simultaneously
- **Multiverse**: Access multiple versions for gradual rollouts
- **Edge caching**: Templates cached globally for fast loading
- **Zero-downtime updates**: Update templates without restarting workers

### Prompt ðŸ¤–

AI prompts, system messages, and instructions for LLM-powered applications.

**Use For**:
- AI agent system prompts (customer support, sales bots)
- Think member prompts (data analysis, content generation)
- Chat templates (conversation starters, guides)
- Few-shot learning examples
- Prompt engineering experiments

**Registration**:
```bash
edgit components add company-analysis prompts/company-analysis.md prompt
edgit components add support-agent prompts/support.md prompt
edgit components add summarizer prompts/summarize.md prompt
```

**Common Metadata**:
```json
{
  "type": "prompt",
  "path": "prompts/company-analysis.md",
  "description": "Analyze company data and generate insights",
  "metadata": {
    "provider": "anthropic",
    "model": "claude-3-5-sonnet-20241022",
    "temperature": 0.7,
    "maxTokens": 4096,
    "variables": ["company_name", "industry", "data"],
    "outputFormat": "structured-json",
    "usedBy": ["analyze-company-member"],
    "category": "analysis"
  }
}
```

**Example Component**:
```markdown prompts/company-analysis.md
You are an expert business analyst specializing in market research.

Analyze the following company data:
- Company: {{company_name}}
- Industry: {{industry}}
- Data: {{data}}

Provide a detailed analysis covering:
1. Market position and competitive advantages
2. Growth opportunities and risks
3. Financial health indicators
4. Strategic recommendations

Format your response as structured JSON with clear sections.
```

**Conductor Integration**:
Prompts power [Conductor's Think members](/conductor/member-types/think). Reference prompts using the `prompt://` protocol:

```yaml
members:
  - name: analyze-company
    type: Think
    config:
      prompt: prompt://company-analysis@v2.0.0
      provider: anthropic
      model: claude-3-5-sonnet-20241022
      data:
        company_name: ${input.companyName}
        industry: ${input.industry}
```

**Why Prompts in Edgit?**
- **Rapid iteration**: Test prompt improvements without redeploying code
- **Version control**: Track prompt evolution and A/B test variations
- **Collaboration**: Share prompts across teams with version history
- **Instant deployment**: Update AI behavior in production immediately
- **Multiverse testing**: Run multiple prompt versions simultaneously

### Function

Serverless functions, utilities, and callable code.

**Use For**:
- AWS Lambda functions
- Cloudflare Workers
- Vercel Functions
- Utility libraries
- API endpoints

**Registration**:
```bash
edgit components add my-function src/functions/my-function.ts function
```

**Common Metadata**:
```json
{
  "type": "function",
  "path": "src/functions/my-function.ts",
  "description": "Processes user authentication",
  "metadata": {
    "runtime": "node18",
    "memory": "256MB",
    "timeout": "30s",
    "triggers": ["http", "queue"],
    "environment": {
      "DATABASE_URL": "required",
      "API_KEY": "required"
    }
  }
}
```

**Example Component**:
```typescript src/functions/auth.ts
export async function authenticate(token: string) {
  // Authentication logic
  return { user: {}, valid: true };
}
```

### Agent

AI agents, bots, and autonomous systems.

**Use For**:
- Customer support bots
- Sales automation agents
- Data processing agents
- AI assistants
- Autonomous workflows

**Registration**:
```bash
edgit components add support-bot src/agents/support.ts agent
```

**Common Metadata**:
```json
{
  "type": "agent",
  "path": "src/agents/support.ts",
  "description": "Customer support AI agent",
  "metadata": {
    "provider": "openai",
    "model": "gpt-4",
    "maxTokens": 2000,
    "temperature": 0.7,
    "systemPrompt": "You are a helpful customer support agent",
    "tools": ["knowledge-base", "ticket-system"]
  }
}
```

**Example Component**:
```typescript src/agents/support.ts
export const supportAgent = {
  model: "gpt-4",
  systemPrompt: "You are a helpful customer support agent",
  async handleMessage(message: string) {
    // Agent logic
    return { response: "..." };
  }
};
```

### Config

Configuration files, settings, and environment variables.

**Use For**:
- Application configs
- Database settings
- API configurations
- Feature flags
- Environment variables

**Registration**:
```bash
edgit components add app-config config/app.json config
```

**Common Metadata**:
```json
{
  "type": "config",
  "path": "config/app.json",
  "description": "Application configuration",
  "metadata": {
    "environment": "production",
    "format": "json",
    "schema": "config/schema.json",
    "encrypted": false,
    "reloadable": true
  }
}
```

**Example Component**:
```json config/app.json
{
  "database": {
    "host": "localhost",
    "port": 5432
  },
  "features": {
    "newUI": true,
    "beta": false
  }
}
```

### SQL

SQL queries, database schemas, and migrations.

**Use For**:
- SQL queries
- Database schemas
- Migrations
- Stored procedures
- Views and indexes

**Registration**:
```bash
edgit components add user-query queries/users.sql sql
```

**Common Metadata**:
```json
{
  "type": "sql",
  "path": "queries/users.sql",
  "description": "User data queries",
  "metadata": {
    "database": "postgres",
    "schema": "public",
    "tables": ["users", "profiles"],
    "readonly": false
  }
}
```

**Example Component**:
```sql queries/users.sql
-- Get active users
SELECT id, email, created_at
FROM users
WHERE active = true
ORDER BY created_at DESC;
```

## Custom Component Types

Create your own component types:

```bash
# Register custom type
edgit components add my-workflow workflows/order.yaml workflow

# Register another custom type
edgit components add my-template templates/email.hbs template
```

Custom types automatically appear in the registry:

```json .edgit/components.json
{
  "components": {
    "workflows": {
      "my-workflow": {
        "type": "workflow",
        "path": "workflows/order.yaml"
      }
    },
    "templates": {
      "my-template": {
        "type": "template",
        "path": "templates/email.hbs"
      }
    }
  }
}
```

## Type-Specific Workflows

### Templates

```bash
# Add template
edgit components add welcome-email templates/email/welcome.html template

# Preview template locally
npm run preview:template welcome-email

# Version
edgit tag create welcome-email v1.0.0

# Deploy
edgit deploy set welcome-email v1.0.0 --to production

# A/B test new version
edgit tag create welcome-email v2.0.0
edgit deploy set welcome-email v2.0.0 --to staging
# Test both versions simultaneously
# template://welcome-email@v1.0.0 vs template://welcome-email@v2.0.0
```

### Prompts

```bash
# Add prompt
edgit components add company-analysis prompts/company-analysis.md prompt

# Test prompt locally with sample data
npm run test:prompt company-analysis

# Version
edgit tag create company-analysis v1.0.0

# Deploy
edgit deploy set company-analysis v1.0.0 --to production

# Iterate on prompt improvements
edgit tag create company-analysis v1.1.0
edgit deploy set company-analysis v1.1.0 --to production
# Zero-downtime: v1.0.0 still available during rollout
```

### Functions

```bash
# Add function
edgit components add payment-processor src/functions/payment.ts function

# Develop and test
npm test -- payment

# Version
edgit tag create payment-processor v1.0.0

# Deploy
edgit deploy set payment-processor v1.0.0 --to production
```

### Agents

```bash
# Add agent
edgit components add support-agent src/agents/support.ts agent

# Test agent locally
npm run test:agent support-agent

# Version
edgit tag create support-agent v1.0.0

# Deploy
edgit deploy set support-agent v1.0.0 --to production
```

### Configs

```bash
# Add config
edgit components add app-config config/app.json config

# Validate config
npm run validate:config config/app.json

# Version
edgit tag create app-config v1.0.0

# Deploy
edgit deploy set app-config v1.0.0 --to production
```

### Function Metadata

```json
{
  "metadata": {
    "runtime": "node18",
    "memory": "256MB",
    "timeout": "30s",
    "triggers": ["http", "queue", "cron"],

      "DATABASE_URL": "required"

  }
```

### Agent Metadata

```json
{
  "metadata": {
```

## Type Detection

Edgit can automatically detect component types:

```bash
# Scan for components
edgit discover scan

# Output:
# Found potential components:
#   src/functions/auth.ts (function)
#   src/agents/support.ts (agent)
#   config/app.json (config)
#   queries/users.sql (sql)

# Register discovered components
edgit discover scan --register
```

## Best Practices

<AccordionGroup>
  <Accordion icon="folder-tree" title="Organization">

    - Keep related components together
    - Separate shared code from components

  <Accordion icon="tag" title="Versioning">
