## What is Edgit?

Edgit extends Git with component-aware tooling:

- **Component Registry**: Track functions, agents, and other components in your repository
- **Git Tag Versioning**: Semantic versioning with native Git tags
- **AI-Powered Commits**: Automatically generate descriptive commit messages
- **Deployment Tracking**: Map versions to environments with metadata

## Key Features

<AccordionGroup>
  <Accordion icon="tags" title="Git Tag-Based Versioning">
    Use native Git tags for semantic versioning. No more version bumps in package.json or separate version files.

    ```bash
    edgit tag email-agent v1.2.0
    git push origin v1.2.0
    ```
  </Accordion>

  <Accordion icon="brain" title="AI-Powered Commit Messages">
    Automatically generate conventional commit messages based on component changes.

    ```bash
    edgit commit  # AI analyzes diffs and generates message
    ```
  </Accordion>

  <Accordion icon="cube" title="Component Registry">
    Track all components in `.edgit/components.json` with metadata, dependencies, and configuration.

    ```json
    {
      "functions": {
        "email-agent": {
          "type": "function",
          "path": "src/agents/email.ts",
          "description": "Email processing agent"
        }
      }
    }
    ```
  </Accordion>

  <Accordion icon="rocket" title="Deployment Metadata">
    Track which versions are deployed to which environments.

    ```bash
    edgit deploy email-agent v1.2.0 --to production
    edgit status email-agent  # Shows deployment history
    ```
  </Accordion>
</AccordionGroup>
